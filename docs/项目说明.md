# tabby-status 项目说明

## 项目定位

`tabby-status` 是一个 Tabby SSH 状态面板插件。插件在 SSH 连接建立后挂载到终端外层布局容器，在窗口左侧展示远程主机状态，包括 CPU、内存、交换、负载、网络速率、延迟、进程、磁盘、IP 和运行时间。

## 核心机制

- 插件入口是 `src/index.ts`，通过 `TerminalDecorator` 挂载到 Tabby 终端标签页。
- 不使用 `sendInput()`，不会把采集命令输入到用户可见终端。
- 通过 Tabby SSH session 的 `openSessionChannel()` 和 `requestExec()` 打开独立 exec channel。
- 每次采集都是一次性命令，不在服务器常驻后台进程。
- 采集结果使用 tab 分隔文本返回，再由 `parseMetricPayload()` 解析。
- `mergeMetricPatch()` 合并局部数据，避免 fast/process/slow 任一采集刷新时清空其他区域。

刷新策略：

- `fast`：每 1 秒刷新 CPU、内存、交换、负载、网络字节计数、Tabby 到 SSH 服务器的 exec 往返延迟。
- `process`：每 3 秒刷新最多 30 条进程。
- `slow`：每 10 秒刷新 IP、运行时间、磁盘列表。

## 平台适配

Tabby 客户端支持 macOS、Windows、Linux：

- 安装路径按客户端系统区分，zip 解压后复制到对应 `plugins/node_modules/tabby-status`。
- 主题、弹窗、菜单、拖拽、复制、SFTP 布局避让等 UI 逻辑不依赖操作系统专有样式。
- 延迟详情在 macOS/Linux 客户端使用本机 `traceroute` / `tracepath`；Windows 客户端使用 PowerShell + `tracert.exe`。
- 本地安装脚本 `scripts/install-local.mjs` 仅用于维护者验证，也按 macOS/Windows/Linux 选择 Tabby 插件目录并复制文件夹。

SSH 目标支持：

- Linux SSH 目标：CPU 使用 `/proc/stat`，内存/交换使用 `/proc/meminfo`，网络使用 `/sys/class/net`，进程使用 `ps`，磁盘使用 `df -P -h`。
- macOS/Darwin SSH 目标：CPU 使用 `top`，内存使用 `vm_stat`，网络使用 `netstat -ibn`，运行时间使用 `sysctl`，进程使用 `ps`。

不支持的场景：

- Tabby 本地默认终端窗口目前不会采集本机系统数据，因为它没有 SSH exec channel。

## UI 行为

- 面板固定在左侧，右边缘可拖拽调整宽度，宽度保存在 `localStorage`。
- 面板、详情弹窗、进程右键菜单和网卡菜单使用 Tabby 当前主题变量派生背景、文字、边框和滚动条颜色。
- 状态面板挂载在终端节点的外层布局容器；终端内容区和 SFTP 面板按同一个状态栏宽度让出空间。
- IP 行只显示 IP，不在 IP 下方展示连接中、等待或采集错误等暗文信息；点击复制后按钮会显示成功或失败反馈。
- CPU、内存、交换行使用固定的“标签 / 进度条 / 辅助数值”布局，进度条内展示百分比和容量信息。
- 进程列表的“内存”和“CPU”表头支持点击排序。
- 点击进程、网络、延迟区域会打开弹窗二级页；进程和网络详情每秒刷新，延迟详情按路由追踪输出流式展示并保留最终结果。
- 所有详情表列宽都可以拖拽调整，表格支持横向滚动。
- 进程详情支持右键菜单：终止进程、复制 PID、复制名称、复制命令、复制位置。
- 网络区域的网卡名称可点击展开网卡列表并切换采集网卡。
- 磁盘列表展示“可用/大小”，可用值取自 `df` 的可用列。

## IP 显示逻辑

公网 IP 和本机 IP 分开采集：

- `publicIp`：通过外部服务获取，按顺序尝试 `api.ipify.org`、`ifconfig.me`、`icanhazip.com`、`checkip.amazonaws.com`。
- `localIp`：从目标机器的默认网卡或路由里获取。

展示优先级：

1. 公网 IP，标记为“公网”。
2. 原始 `ip` 字段。
3. 本机网卡 IP，标记为“本机”。

## 安装

只保留 zip 安装这一种方式。

1. 从 GitHub Actions Artifacts 或 GitHub Release 下载 `tabby-status-<version>.zip`。
2. 完全退出 Tabby。
3. 解压 zip，得到 `tabby-status` 文件夹。
4. 把 `tabby-status` 文件夹复制到对应系统的 Tabby 插件目录。
5. 重启 Tabby。

macOS：

```text
~/Library/Application Support/tabby/plugins/node_modules/tabby-status
```

Windows：

```text
%APPDATA%\tabby\plugins\node_modules\tabby-status
```

Linux：

```text
~/.config/tabby/plugins/node_modules/tabby-status
```

最终目录必须是：

```text
plugins/node_modules/tabby-status/package.json
plugins/node_modules/tabby-status/dist/index.js
```

不能多套一层 `tabby-status/tabby-status`。

- GitHub Actions 只上传/发布 zip。
- zip 打包脚本不会生成 tgz。

## 已知限制

- 公网 IP 需要目标机器能访问外部 IP 服务；如果出站网络、DNS、代理或防火墙限制导致访问失败，会回退显示本机 IP。
- Windows 客户端的延迟详情依赖系统自带 `tracert.exe`，地理位置补充能力弱于 macOS/Linux 的 traceroute 路径。
- Tabby SSH session 字段属于插件环境内部接口，未来 Tabby 大版本变更时可能需要适配。
