## Autonomy

- 对已授权任务持续执行到完成，不要频繁停止 需要我说“继续”。
- 能合理推断的地方直接选择保守方案并继续。
- 长任务分批执行，简短汇报进度后继续下一批。
- 只有在会删除/覆盖重要数据、缺少凭据、需要付款/登录、需求冲突或明显高风险时才停下来问我。

## Release Gate

- 代码实现完成后，必须先在本地完成构建、测试和 Tabby 本地安装。
- 用户本地验收前，禁止执行 `git push`、推送 tag、创建 GitHub Release 或触发远端发布。
- 只有用户明确说可以 `push` 或要求发布时，才允许推送代码、tag 或 Release。
